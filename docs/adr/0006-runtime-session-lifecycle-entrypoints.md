# Runtime session lifecycle entrypoints

The runtime will model execution around session lifecycle rather than the ambiguous one-shot/resumable split: ephemeral execution does not prepare provider-session continuity, new-session execution prepares and returns consumer-owned continuation state, and resumed-session execution consumes that continuation without service fallback. These lifecycle entrypoints replace the previous canonical one-shot/resumable API split. ToolPolicy is an independent explicit value that defaults to no tools, workspace access belongs under tool-policy configuration, and provider-session continuity keeps one `ToolPolicy` while allowing resumed model or effort overrides. The runtime remains stateless between calls: consumers store or discard continuations, provider adapters interpret provider resume state, and expected orchestration conditions such as usage limits, cancellation, timeout, temporary service unavailability, and confidently retryable provider failures are normal canonical outcomes carrying invocation progress rather than exceptional failures.
